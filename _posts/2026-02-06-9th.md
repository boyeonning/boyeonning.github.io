# Python @dataclass 완벽 이해하기

## @dataclass란?

`@dataclass`는 Python 3.7부터 추가된 데코레이터로, **데이터를 저장하는 클래스를 쉽게 만들 수 있게 해주는 도구**입니다.

## 왜 써야 할까요?

### 1. 코드가 엄청 짧아집니다

**기존 방식 (불편함 😓)**
```python
class Person:
    def __init__(self, name, age, email):
        self.name = name
        self.age = age
        self.email = email
    
    def __repr__(self):
        return f"Person(name={self.name}, age={self.age}, email={self.email})"
    
    def __eq__(self, other):
        if not isinstance(other, Person):
            return False
        return (self.name == other.name and 
                self.age == other.age and 
                self.email == other.email)
```

**@dataclass 사용 (편함 😊)**
```python
from dataclasses import dataclass

@dataclass
class Person:
    name: str
    age: int
    email: str
```

위 두 코드는 **완전히 같은 기능**을 합니다! 하지만 dataclass는 훨씬 짧고 읽기 쉽습니다.

### 2. 자동으로 만들어주는 것들

`@dataclass`를 붙이면 다음 메서드들이 **자동으로 생성**됩니다:

- `__init__()`: 초기화 메서드
- `__repr__()`: 객체를 출력할 때 보기 좋게 표시
- `__eq__()`: 객체 비교 (==)
- 그 외 여러 편의 메서드들

### 3. 타입 힌트가 명확합니다

```python
@dataclass
class Product:
    name: str          # 문자열이어야 함
    price: int         # 정수여야 함
    in_stock: bool     # 참/거짓이어야 함
```

변수 타입이 한눈에 보여서 코드를 이해하기 쉽습니다.

## 실전 예제

### 기본 사용법

```python
from dataclasses import dataclass

@dataclass
class Book:
    title: str
    author: str
    price: int
    pages: int

# 객체 생성
book1 = Book("파이썬 기초", "홍길동", 25000, 350)
book2 = Book("파이썬 기초", "홍길동", 25000, 350)

# 자동으로 생성된 __repr__
print(book1)
# 출력: Book(title='파이썬 기초', author='홍길동', price=25000, pages=350)

# 자동으로 생성된 __eq__
print(book1 == book2)  # True
```

### 기본값 설정

```python
@dataclass
class Config:
    host: str = "localhost"
    port: int = 8080
    debug: bool = False

# 기본값 사용
config1 = Config()
print(config1)  # Config(host='localhost', port=8080, debug=False)

# 일부만 변경
config2 = Config(port=9000)
print(config2)  # Config(host='localhost', port=9000, debug=False)
```

### 변경 불가능한 객체 만들기

```python
@dataclass(frozen=True)
class Point:
    x: int
    y: int

point = Point(10, 20)
# point.x = 30  # 에러! frozen=True로 설정했기 때문에 변경 불가
```

### 리스트나 딕셔너리 기본값

```python
from dataclasses import dataclass, field
from typing import List

@dataclass
class Student:
    name: str
    grades: List[int] = field(default_factory=list)
    # default_factory를 사용해야 안전합니다!

student1 = Student("김철수")
student1.grades.append(90)

student2 = Student("이영희")
print(student1.grades)  # [90]
print(student2.grades)  # [] - 독립적인 리스트
```

## 주요 옵션들

```python
@dataclass(
    init=True,          # __init__ 자동 생성 (기본값: True)
    repr=True,          # __repr__ 자동 생성 (기본값: True)
    eq=True,            # __eq__ 자동 생성 (기본값: True)
    order=False,        # __lt__, __le__ 등 비교 메서드 생성 (기본값: False)
    frozen=False,       # 불변 객체로 만들기 (기본값: False)
)
class MyClass:
    field1: str
    field2: int
```

## 실무 활용 예제

### 데이터베이스 모델

```python
from dataclasses import dataclass
from datetime import datetime

@dataclass
class User:
    user_id: int
    username: str
    email: str
    created_at: datetime
    is_active: bool = True

# 사용
user = User(
    user_id=1,
    username="ugly_potato",
    email="potato@example.com",
    created_at=datetime.now()
)
```

### API 응답 처리

```python
from dataclasses import dataclass
from typing import List

@dataclass
class WeatherData:
    temperature: float
    humidity: int
    description: str
    wind_speed: float

@dataclass
class WeatherResponse:
    city: str
    country: str
    data: WeatherData
    
# JSON을 dataclass로 쉽게 변환 가능
```

## 언제 사용하면 좋을까요?

✅ **이럴 때 사용하세요:**
- 데이터를 담는 클래스를 만들 때
- 설정(Config) 클래스를 만들 때
- API 요청/응답 모델을 만들 때
- 데이터베이스 모델을 만들 때
- 여러 값을 묶어서 전달할 때

❌ **이럴 때는 사용하지 마세요:**
- 복잡한 비즈니스 로직이 많은 클래스
- 상속 구조가 복잡한 경우
- 동적으로 속성을 추가/제거해야 하는 경우

## 정리

`@dataclass`는:
1. **코드를 짧고 깔끔하게** 만들어줍니다
2. **반복 작업을 줄여줍니다** (boilerplate 코드 제거)
3. **타입 힌트로 가독성**을 높여줍니다
4. **실수를 줄여줍니다** (자동 생성으로 버그 감소)

데이터를 다루는 클래스를 만든다면 `@dataclass`를 사용하세요! 코드가 훨씬 깔끔해집니다. 🚀
