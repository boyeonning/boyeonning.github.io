## 🧠 핵심 비유: 커피숍 바리스타

### 🔁 비동기 (Asynchronous)

**"혼자 일하지만 기다리는 시간을 낭비하지 않는 바리스타"**

```
바리스타 1명, 손님 3명

1. 손님 A 주문 받기 → 에스프레소 추출 시작 (30초 기다려야 함)
2. ↪ 기다리는 동안 손님 B 주문 받기 → 우유 스팀 시작
3. ↪ 기다리는 동안 손님 C 주문 받기
4. 에스프레소 완성 → 손님 A 음료 완성
5. 우유 완성 → 손님 B 음료 완성
```

- **직원(스레드)은 1명** — 동시에 여러 일을 하는 게 아니라, **기다리는 틈틈이** 다른 일을 처리
- I/O 작업(네트워크, 파일 읽기 등) 대기 시간을 효율적으로 활용
- Python `asyncio`, JavaScript `Promise/await`

---

### 🧵 멀티스레드 (Multi-thread)

**"같은 주방에서 일하는 바리스타 여러 명"**

```
바리스타 3명, 같은 주방 공유

바리스타 A: 손님 1 음료 제조 중
바리스타 B: 손님 2 음료 제조 중  ← 동시에!
바리스타 C: 손님 3 음료 제조 중

⚠️ 문제: 설탕통이 1개 → 동시에 집으려다 충돌!
```

- **같은 메모리(주방) 공유** → 빠른 데이터 공유 가능
- 하지만 **공유 자원 충돌(Race Condition)** 위험
- I/O 처리에 적합, CPU 집중 작업은 Python GIL로 제한
- Python `threading`, Java `Thread`

---

### 🏭 멀티프로세스 (Multi-process)

**"각자 독립된 주방을 가진 바리스타들"**

```
주방 A (프로세스 1): 바리스타 독립 운영
주방 B (프로세스 2): 바리스타 독립 운영  ← 완전 독립!
주방 C (프로세스 3): 바리스타 독립 운영

✅ 충돌 없음! 각자 자기 설탕통, 자기 도구 사용
⚠️ 단점: 주방 차리는 비용(메모리)이 큼
```

- **독립된 메모리** → 충돌 없이 진짜 병렬 처리 가능
- Python GIL 우회 → **CPU 집중 작업에 최적**
- 메모리 소비가 크고 프로세스 간 통신이 복잡
- Python `multiprocessing`

## 📊 한눈에 비교

| 항목 | 비동기 | 멀티스레드 | 멀티프로세스 |
|------|--------|-----------|------------|
| **직원 수** | 1명 | 여러 명 (공유 주방) | 여러 명 (독립 주방) |
| **메모리** | 공유 | 공유 | 독립 |
| **진짜 병렬?** | ❌ (번갈아 처리) | △ (GIL 제한) | ✅ |
| **적합한 작업** | I/O 대기 많을 때 | I/O + 가벼운 병렬 | CPU 집중 작업 |
| **오버헤드** | 낮음 | 중간 | 높음 |
| **충돌 위험** | 낮음 | 있음 | 없음 |

## 🎯 언제 뭘 써야 할까?

```
📡 웹 크롤링, API 요청, DB 쿼리
   → 비동기 (asyncio) 또는 멀티스레드

🖼️ 이미지 처리, 영상 인코딩, 머신러닝 연산
   → 멀티프로세스

💬 채팅 서버, 실시간 알림
   → 비동기 (이벤트 루프)
```

---

## 💡 한 줄 요약

- **비동기**: 혼자서 눈치껏 빈틈을 채운다
- **멀티스레드**: 같은 집에 사는 룸메이트들이 각자 일한다
- **멀티프로세스**: 각자 집을 차려서 완전히 독립적으로 일한다
