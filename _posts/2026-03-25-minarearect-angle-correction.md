## 코드

```python
angle = cv2.minAreaRect(coords)[-1]

if angle < -45:
    angle = 90 + angle
```

## `minAreaRect`가 하는 일

어두운 픽셀들을 감싸는 **최소 면적 회전 사각형**을 구한다.

반환값은 `(중심점, (너비, 높이), 각도)` 형태이며, `[-1]`로 각도만 추출한다.

## 각도는 어떻게 측정되나?

`minAreaRect`는 사각형의 **"width" 변**(첫 번째 차원)이 **수평 x축**과 이루는 각도를 반환한다.

범위는 항상 **-90° ~ 0°** 이다.

## 문제: 기준 변이 뒤바뀐다

여기가 핵심이다. OpenCV는 어떤 변을 "width"로 잡을지가 **일관되지 않다**.

두 경우 모두 실제로는 문서가 10도 정도만 기울어져 있다. 하지만 OpenCV가 기준 변을 다르게 잡으면서 전혀 다른 각도가 나온다.

### Case 1: angle = -10 (보정 불필요)

![Case 1: angle = -10](/assets/images/case1.svg)

- "width" = **긴 변** (가로 방향)
- 긴 변이 수평에서 10도 기울어짐 → angle = **-10**
- `-10 > -45` 이므로 `if`문 진입 안 함
- **-10이 곧 실제 기울기** ✅

### Case 2: angle = -80 (보정 필요!)

![Case 2: angle = -80](/assets/images/case2.svg)

- "width" = **짧은 변** (세로 방향에 가까움)
- 짧은 변이 수평에서 80도 기울어짐 → angle = **-80**
- 하지만 실제 문서는 **거의 똑바로 서 있고, 살짝 10도 기울어진 것뿐!**
- `-80 < -45` 이므로 `if`문 진입
- `90 + (-80) = 10` → **실제 기울기 10도** ✅

## 보정 요약

![보정 요약](/assets/images/summary.svg)

| 각도 범위 | 의미 | 보정 |
|-----------|------|------|
| `-45 < angle ≤ 0` | "width"가 긴 변 (가로에 가까움) | 그대로 사용 |
| `-90 ≤ angle ≤ -45` | "width"가 짧은 변 (세로에 가까움) | `90 + angle` |

## 왜 -45가 기준인가?

-45°는 정사각형이 되는 지점, 즉 가로와 세로의 구분이 없어지는 경계이다.

이 경계를 기준으로 OpenCV가 "width"로 잡는 변이 **뒤바뀌기 때문에** -45를 분기점으로 사용한다.

## 한 줄 요약

> `minAreaRect`는 기준 변을 제멋대로 잡아서 엉뚱한 각도를 줄 수 있다.
> `if angle < -45: angle = 90 + angle`는 이걸 보정해서 **실제 문서 기울기**를 얻는 코드다.
