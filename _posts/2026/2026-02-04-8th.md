# Pydantic Field 쉽게 이해하기

## Field가 뭔가요?

Pydantic의 `Field`는 클래스의 속성(필드)에 **추가 정보와 검증 규칙**을 붙이는 도구입니다.

쉽게 말해서:
- 일반 변수: 그냥 값만 저장
- Field를 쓴 변수: 값 + 설명 + 검증 규칙 + 기본값 등등

---

## 왜 사용하나요?

### 1. **기본값 설정**
```python
from pydantic import BaseModel, Field

class User(BaseModel):
    name: str
    age: int = Field(default=0)  # 나이를 안 넣으면 자동으로 0
    is_active: bool = Field(default=True)  # 기본값 True
```

### 2. **값 검증 (유효성 검사)**
```python
class Product(BaseModel):
    name: str = Field(min_length=1, max_length=50)  # 1~50자만 허용
    price: int = Field(gt=0, le=1000000)  # 0보다 크고, 100만 이하
    stock: int = Field(ge=0)  # 0 이상만 허용
```

### 3. **설명 추가**
```python
class User(BaseModel):
    email: str = Field(description="사용자 이메일 주소")
    phone: str = Field(description="연락처 (하이픈 없이)")
```

---

## 주요 파라미터

### 기본값 관련
```python
age: int = Field(default=20)  # 기본값
name: str = Field(default_factory=lambda: "익명")  # 함수로 기본값 생성
```

### 검증 관련
```python
# 숫자 검증
price: int = Field(gt=0)        # greater than: 0보다 큼
price: int = Field(ge=0)        # greater or equal: 0 이상
price: int = Field(lt=100)      # less than: 100 미만
price: int = Field(le=100)      # less or equal: 100 이하

# 문자열 검증
name: str = Field(min_length=2, max_length=30)  # 길이 제한
email: str = Field(pattern=r'^[\w\.-]+@[\w\.-]+\.\w+$')  # 정규식 패턴

# 리스트/컬렉션 검증
tags: list = Field(min_length=1, max_length=5)  # 최소 1개, 최대 5개
```

### 문서화 관련
```python
name: str = Field(description="사용자 이름")
age: int = Field(title="나이", description="만 나이")
```

## 실전 예제

### 예제 1: 회원가입 정보
```python
from pydantic import BaseModel, Field
from typing import Optional

class SignupForm(BaseModel):
    username: str = Field(
        min_length=3,
        max_length=20,
        description="아이디 (3~20자)"
    )
    password: str = Field(
        min_length=8,
        description="비밀번호 (최소 8자)"
    )
    age: int = Field(
        ge=14,
        le=120,
        description="만 14세 이상"
    )
    email: str = Field(
        pattern=r'^[\w\.-]+@[\w\.-]+\.\w+$',
        description="이메일 주소"
    )
    nickname: Optional[str] = Field(
        default=None,
        max_length=30,
        description="닉네임 (선택사항)"
    )

# 사용 예시
user = SignupForm(
    username="hong123",
    password="secure1234!",
    age=25,
    email="hong@example.com"
)
print(user.model_dump())
```

### 예제 2: 상품 정보
```python
from datetime import datetime

class Product(BaseModel):
    product_id: str = Field(
        pattern=r'^PRD-\d{6}$',
        description="상품 ID (예: PRD-000001)"
    )
    name: str = Field(
        min_length=1,
        max_length=100,
        description="상품명"
    )
    price: int = Field(
        gt=0,
        le=10000000,
        description="가격 (1원 ~ 1000만원)"
    )
    discount_rate: float = Field(
        ge=0.0,
        le=1.0,
        default=0.0,
        description="할인율 (0.0 ~ 1.0)"
    )
    stock: int = Field(
        ge=0,
        default=0,
        description="재고 수량"
    )
    is_available: bool = Field(
        default=True,
        description="판매 가능 여부"
    )
    created_at: datetime = Field(
        default_factory=datetime.now,
        description="등록 시간"
    )

# 사용
product = Product(
    product_id="PRD-000001",
    name="노트북",
    price=1500000,
    discount_rate=0.15,
    stock=10
)
```

## Field vs 일반 기본값

```python
# 일반 기본값
class User1(BaseModel):
    age: int = 0  # 단순 기본값만

# Field 사용
class User2(BaseModel):
    age: int = Field(default=0, ge=0, le=150)  # 기본값 + 검증

# Field가 더 안전!
try:
    user = User1(age=-5)  # 통과됨 😱
    print(user.age)  # -5
except:
    pass

try:
    user = User2(age=-5)  # 에러 발생! ✅
except Exception as e:
    print(f"검증 실패: {e}")
```

## 자주 쓰는 패턴

### 1. 필수값 vs 선택값
```python
from typing import Optional

class UserProfile(BaseModel):
    # 필수값 (기본값 없음)
    username: str = Field(min_length=3)
    
    # 선택값 (None 가능)
    bio: Optional[str] = Field(default=None, max_length=500)
    
    # 선택값 (기본값 있음)
    profile_image: str = Field(default="default.jpg")
```

### 2. 숫자 범위 체크
```python
class Score(BaseModel):
    math: int = Field(ge=0, le=100, description="수학 점수 (0~100)")
    english: int = Field(ge=0, le=100, description="영어 점수 (0~100)")
    
    def average(self) -> float:
        return (self.math + self.english) / 2
```

### 3. 문자열 패턴 검증
```python
class Contact(BaseModel):
    phone: str = Field(
        pattern=r'^\d{2,3}-\d{3,4}-\d{4}$',
        description="전화번호 (예: 010-1234-5678)"
    )
    zipcode: str = Field(
        pattern=r'^\d{5}$',
        description="우편번호 (5자리)"
    )
```

## 핵심 정리

| 기능 | 설명 | 예시 |
|------|------|------|
| `default` | 기본값 설정 | `Field(default=0)` |
| `gt`, `ge`, `lt`, `le` | 숫자 범위 검증 | `Field(ge=0, le=100)` |
| `min_length`, `max_length` | 길이 검증 | `Field(min_length=1, max_length=50)` |
| `pattern` | 정규식 검증 | `Field(pattern=r'^\d+$')` |
| `description` | 필드 설명 | `Field(description="사용자 이름")` |

**언제 쓰나요?**
- ✅ 입력값을 검증하고 싶을 때
- ✅ 기본값을 설정하고 싶을 때
- ✅ API 문서를 자동 생성하고 싶을 때
- ✅ 데이터 구조를 명확히 정의하고 싶을 때

**안 써도 되는 경우는?**
- 단순히 타입만 정의하면 되는 경우
- 검증이 필요 없는 내부 클래스

---

## 마무리

Pydantic Field는 **"이 값은 이런 조건을 만족해야 해!"** 라고 선언하는 도구입니다.

한 번만 정의해두면:
- 자동으로 검증해주고
- 잘못된 값은 에러를 내주고
- 문서도 자동으로 만들어집니다

**코드가 더 안전하고 명확해집니다!** 🎯
