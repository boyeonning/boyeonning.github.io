## 들어가며

`cv2.minAreaRect`로 문서의 기울기를 구하다 보면, 이런 보정 코드를 자주 만나게 된다.

```python
if angle < -45:
    angle = 90 + angle
```

처음 보면 _왜 90을 더하지?_, _-45는 뭔데?_ 싶다.  
이 글에서 그 이유를 처음부터 천천히 풀어본다.

---

## 1. 직사각형에는 변이 두 종류다

너무 당연하지만, 여기서부터 시작해야 한다.

```
가로로 누운 경우:

  긴 변 → ┌──────────────────┐
          │                  │ ← 짧은 변
  긴 변 → └──────────────────┘

세로로 세운 경우:

          ┌──────┐
   긴 변 →│      │← 긴 변
          │      │
          │      │
          └──────┘
           ↑ 짧은 변
```

- **가로로 누우면** → 긴 변이 위아래, 짧은 변이 좌우
- **세로로 세우면** → 긴 변이 좌우, 짧은 변이 위아래

종이를 돌리면 **어떤 변이 "아래쪽"에 오느냐**가 바뀐다.  
이게 뒤에서 전부 중요해진다.

---

## 2. "꼭짓점에서 각도를 잰다"는 무슨 뜻인가

시계를 떠올려보자.  
시계 중심에서 3시 방향(→)과 시계바늘 사이가 얼마나 벌어졌는지 — 그게 "각도"다.

OpenCV도 똑같다.

```
              변 (시계바늘)
             ╱
            ╱
           ╱  ← 이 벌어진 정도 = 각도
꼭짓점 ●──────────→ 수평선 (3시 방향)
```

**꼭짓점** = 기준점 (시계의 중심)  
**수평선** = 3시 방향 (→)  
**변** = 시계바늘  

"꼭짓점에서 각도를 잰다" = **모서리 한 점을 중심으로, 수평선과 변 사이가 얼마나 벌어졌는지 측정한다**.

---

## 3. 꼭짓점은 어디인가

직사각형에는 꼭짓점이 4개 있다.  
OpenCV는 그중 **y 좌표가 가장 큰 점**, 즉 화면에서 **가장 아래에 있는 모서리**를 기준점으로 잡는다.

```
        B ●─────────────● C
         ╱              ╱
        ╱              ╱
    A ●─────────────● D  ← y가 가장 큼 = 가장 아래
                             OpenCV는 여기를 기준으로 잡는다
```

참고로 직선 위 어디서 수평선을 긋든 기울기는 같다.  
꼭짓점을 고른 건 "대표 점이 필요해서"일 뿐이다.

---

## 4. 시계방향이면 마이너스(-)

가장 아래 꼭짓점에서 수평선(→)을 긋고, 거기서 변까지의 방향을 본다.

- **반시계방향(위쪽)으로 가야 하면 → +**
- **시계방향(아래쪽)으로 가야 하면 → -**

```
          변 (반시계 = +)
         ╱
        ╱  +30°
꼭짓점 ●──────────→ 수평선
        ╲  -30°
         ╲
          변 (시계 = -)
```

`minAreaRect`는 변이 항상 수평선 위쪽으로 가기 때문에, 시계방향으로 재면 결과는 항상 **-90° ~ 0°** 사이다.

---

## 5. 같은 종이인데 답이 두 개?

세로 A4를 5도 기울여서 놓았다고 하자.

**해석 A — 짧은 변 기준**

짧은 변(가로)이 아래에 있다고 보면, 수평선과 거의 일치하니까:

```
꼭짓점 ●━━━━━━━→  짧은 변
        ──────→  수평선
        거의 같다 → angle = -5°
```

**해석 B — 긴 변 기준**

긴 변(세로)이 아래에 있다고 보면, 수평선에서 거의 수직까지 돌아야 하니까:

```
        │ 긴 변 (거의 수직)
        │
        │  ← 여기까지 거의 90° 돌아야 함
꼭짓점 ●──────→ 수평선
        
        → angle = -85°
```

**똑같은 종이를 보고 있는데, 어떤 변을 "아래쪽"으로 잡느냐에 따라 -5° 또는 -85°가 나온다.**

OpenCV는 이걸 우리 마음대로 정해주지 않는다.  
그래서 보정이 필요하다.

---

## 6. 왜 -45°가 경계인가

`minAreaRect`는 항상 **-90° ~ 0°** 범위의 값을 반환한다.

```
-90°                    -45°                     0°
  |━━━━ 보정 필요 ━━━━━━|━━━━ 그대로 사용 ━━━━━━|
  (긴 변 기준으로 잰 것)   (짧은 변 기준으로 잰 것)
```

| 구간 | 의미 | 조치 |
|------|------|------|
| **-45° ~ 0°** | 짧은 변(가로) 기준으로 잰 것 | ✅ 그대로 사용 |
| **-90° ~ -45°** | 긴 변(세로) 기준으로 잰 것 | ⚠️ 보정 필요 |

-45°는 직사각형이 정확히 대각선으로 기울어지는 지점이다.  
이 각도를 넘어가는 순간, "이건 가로 문서가 많이 기울어진 게 아니라, **세로 문서를 긴 변 기준으로 잰 거다**"라고 판단할 수 있다.

---

## 7. 왜 90을 더하면 실제 기울기가 나오는가

이건 수학이 아니라 **직사각형의 성질** 때문이다.

직사각형의 꼭짓점 각도는 항상 **90°**다.

```
        │ 긴 변 (β)
        │
        │  ← 꼭짓점 (항상 90°)
        └──────── 짧은 변 (α)
        ─ ─ ─ ─ ─ 수평선
```

- **α** = 짧은 변과 수평선 사이 각도 = **우리가 원하는 값**
- **β** = 긴 변과 수평선 사이 각도 = **OpenCV가 준 값**

> **α + β = 90°**  
> **α = 90° - β = 90° + (-β) = 90 + angle**

구체적으로:

```python
# OpenCV가 -85°를 반환했다면
angle = -85
angle = 90 + (-85)  # = 5°
# 실제 기울기인 5°로 복원!
```

---

## 8. 전체 보정 코드

```python
rect = cv2.minAreaRect(contour)
angle = rect[2]  # -90 ~ 0 범위

if angle < -45:
    angle = 90 + angle
    # 긴 변 기준 → 짧은 변 기준으로 변환

# 이제 angle은 실제 문서의 기울기
```

## 정리

| 질문 | 답 |
|------|-----|
| 꼭짓점이 뭔데? | 직사각형의 모서리 점. OpenCV는 가장 아래 점을 고른다 |
| 거기서 뭘 하는 건데? | 수평선(→)을 긋고, 변까지 벌어진 정도를 잰다 |
| 왜 마이너스인가? | 시계방향 회전 = 음수 |
| 왜 엉뚱한 값이 나오나? | OpenCV가 긴 변을 기준으로 잡을 수 있어서 |
| 왜 -45°가 경계인가? | 가로/세로 구분이 뒤집히는 지점 |
| 왜 90을 더하면 되나? | 긴 변 각도 + 짧은 변 각도 = 90° (직각) |

> **한 줄 요약:** OpenCV가 가로/세로 중 어떤 변을 기준으로 잴지 모르니까, -45° 기준으로 판별해서 90°를 더해 보정한다.
